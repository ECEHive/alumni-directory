# Hive Alumni Directory

A static, encrypted alumni directory for Georgia Tech's [Hive Makerspace](https://hive.gatech.edu).

Alumni data is AES-GCM encrypted at build time and stored as a single `.enc` file. The decryption key is never sent to the server — it lives only in the URL fragment (`#your-key`).

---

## Updating the Alumni Data

### Step 1 — Open the import tool

Navigate to the [Data Import](https://alumni.hivemakerspace.com/tools/data-import) page.

You can use the development server if needed:
```
bun install
bun dev
# go to http://localhost:4321/tools/data-import
```

### Step 2 — Import your spreadsheet

Upload a CSV or XLSX file. Map the columns to the alumni fields and click **Generate**. Copy the `ALUMNI_DATA=...` value shown in the output.

### Step 3 — Update the GitHub secret

In the repository go to **Settings → Secrets and variables → Actions** and update the `ALUMNI_DATA` secret with the value you copied.

> The `ALUMNI_SECRET_KEY` secret is the encryption passphrase **and** the URL fragment users need to access the site (e.g. `https://alumni.hivemakerspace.com/#your-key`). Update it here too if you are rotating the key.

### Step 4 — Redeploy

Commit any change or manually trigger the workflow under the **Actions** tab. GitHub Actions will encrypt the data and publish the updated site automatically.

---

## GitHub Actions Setup

Required repository secrets (`Settings → Secrets and variables → Actions`):

| Secret | Description |
|---|---|
| `ALUMNI_SECRET_KEY` | Encryption passphrase — also the URL fragment users append to access the site |
| `ALUMNI_DATA` | Encoded alumni blob generated by the `/tools/data-import` utility |

## Local Development

```sh
bun install   # install dependencies
bun dev       # dev server at localhost:4321
bun build     # production build → dist/
bun preview   # preview the production build locally
```

Copy `.env.example` to `.env` and fill in both values before running locally.
